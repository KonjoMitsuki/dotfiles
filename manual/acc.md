## ✅ 結論（これだけ覚えればOK）

`acc` は
👉 **問題をまとめて取得 → ローカルで解く → テスト → 提出**
を一瞬で回すためのツール。
普段使うのは **5コマンドだけ**。

---

## 🧠 accの基本的な考え方

accは

* コンテスト単位でディレクトリを作る
* 問題ごとにフォルダを分ける
* テンプレを自動配置する

という「**競プロに最適化されたフォルダ管理ツール**」。

---

## 🛠 よく使う5コマンド（超重要）

### ① 問題を取得する

```bash
acc new abc350
```

📂 こうなる：

```
abc350/
 ├─ a/
 │   ├─ main.cpp
 │   └─ tests/
 ├─ b/
 ├─ c/
```

---

### ② 問題フォルダに入る

```bash
cd abc350/a
```

---

### ③ コードを書く

```bash
code main.cpp   # VSCode
# or
vim main.cpp
```

---

### ④ サンプルテストを実行

```bash
oj test
```

成功：

```
[AC] sample-1
[AC] sample-2
```

失敗：

```
[WA] sample-1
```

---

### ⑤ 提出する

```bash
acc submit main.cpp
```

---

## 📌 コマンド一覧（用途別）

### 🔹 コンテスト・問題管理

| コマンド               | 何をする      |
| ------------------ | --------- |
| `acc new abc350`   | コンテスト取得   |
| `acc new abc350 a` | 問題Aだけ取得   |
| `acc add b`        | B問題を後から追加 |
| `acc check`        | 現在位置が正しいか |

---

### 🔹 提出関連

| コマンド                     | 内容      |
| ------------------------ | ------- |
| `acc submit main.cpp`    | 通常提出    |
| `acc submit main.cpp -f` | 確認なし即提出 |

---

### 🔹 設定系

| コマンド         | 内容          |
| ------------ | ----------- |
| `acc login`  | AtCoderログイン |
| `acc logout` | ログアウト       |
| `acc config` | 設定確認        |
| `acc init`   | テンプレ初期化     |

---

## 🧪 oj test の便利オプション

```bash
oj test -c "g++ main.cpp -O2"
```

標準入力を手動で：

```bash
oj test -i
```

---

## 📁 ディレクトリ構成の意味

```
abc350/
 └─ a/
    ├─ main.cpp      ← 提出するコード
    └─ tests/        ← サンプルケース
```

👉 **提出するのは必ず `main.cpp`**

## ⚠️ よくあるミス

### ❌ `acc submit` だけ打つ

```bash
acc submit
```

👉 エラーになる
✅ 正解：

```bash
acc submit main.cpp
```

---

### ❌ 問題フォルダ外で提出

```bash
abc350/
acc submit main.cpp
```

👉 ダメ
✅ `a/` や `b/` に入ってから
